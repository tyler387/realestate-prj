# 커뮤니티탭 아파트 선택 팝업 지도 마커 개발 플랜

## 1. 목적

커뮤니티탭 좌측 아파트 선택 팝업의 `지도` 탭에서 마커가 너무 크거나 겹쳐 보이는 문제를 줄이고, 줌 레벨에 따라 지역 단위 묶음과 개별 아파트 선택을 자연스럽게 전환한다.

최종 목표는 사용자가 지도에서 아파트를 탐색할 때 다음 흐름을 명확히 제공하는 것이다.

1. 넓은 줌에서는 같은 구/지역의 아파트를 묶어서 표시한다.
2. 가까운 줌에서는 개별 아파트 마커를 작고 일관된 스타일로 표시한다.
3. 마커 클릭 시 바로 선택하지 않고 작은 정보 카드에서 한 번 더 확정한다.
4. 확정 시 부모 모달에 아파트 정보를 전달하고 우리아파트 커뮤니티로 이동한다.

## 2. 현재 문제

1. 마커 크기가 커서 지도 정보를 가린다.
2. 가격 라벨이 많아지면 지도에서 시각적 소음이 크다.
3. 같은 구 또는 가까운 지역의 아파트가 밀집되면 마커가 겹친다.
4. 팝업 지도와 메인 지도에서 같은 마커 렌더링 훅을 공유해, 한쪽 수정이 다른 화면에 영향을 줄 수 있다.
5. 마커 클릭과 아파트 최종 선택의 단계가 명확히 분리되어야 한다.

## 3. 개발 방향

### 3.1 마커 표시 정책

팝업 지도에서는 메인 지도보다 정보량을 줄인다.

- 메인 지도: 가격 중심 마커 유지
- 아파트 선택 팝업 지도: 탐색/선택 중심의 작은 마커 사용

팝업 지도 마커는 다음 정보만 우선 표시한다.

- 개별 아파트: 가격 또는 짧은 아파트명 중 하나만 표시
- 지역 묶음: `강남구 24개`, `송파구 18개` 같은 집계 표시

### 3.2 줌 레벨별 표시

1. 넓은 줌
   - 개별 아파트 마커를 표시하지 않는다.
   - 같은 구 또는 행정동 단위로 묶음 마커를 표시한다.
   - 묶음 마커 클릭 시 해당 지역 중심으로 확대한다.

2. 중간 줌
   - 행정동 또는 화면 격자 단위로 묶음 마커를 표시한다.
   - 묶음 수가 적은 곳은 개별 마커로 전환한다.

3. 가까운 줌
   - 개별 아파트 마커를 표시한다.
   - 마커는 작고 동일한 형태로 유지한다.
   - 클릭 시 하단 정보 카드가 뜬다.

## 4. 구현 범위

### 4.1 프론트엔드

대상 파일:

1. `frontend/src/hooks/useMapMarkers.ts`
2. `frontend/src/components/features/apartment-select/MapSection.tsx`
3. `frontend/src/utils/mapMarkerContent.ts`
4. `frontend/src/types/map.ts`
5. 필요 시 신규 파일
   - `frontend/src/utils/mapMarkerGrouping.ts`
   - `frontend/src/utils/popupMapMarkerContent.ts`

작업:

1. 메인 지도 마커와 팝업 지도 마커 렌더링 옵션 분리
2. 팝업 지도 전용 작은 마커 디자인 추가
3. 같은 구/행정동 묶음 마커 디자인 추가
4. 줌 레벨 또는 마커 수 기준으로 묶음/개별 표시 전환
5. 마커 클릭 시 정보 카드 표시
6. 정보 카드의 `이 아파트 선택` 클릭 시 부모 콜백 호출

### 4.2 백엔드/API 검토

현재 `/api/v1/apartments/markers` 응답에 `eupMyeonDong`은 있지만 `sigungu`가 명시적으로 필요할 수 있다.

확인 대상:

1. `backend/src/main/java/com/realestate/web/controller/ApartmentController.java`
2. marker 응답 DTO
3. 지도 마커 조회 서비스

필요 시 추가:

- marker 응답에 `sigungu` 포함
- 프론트 `MapMarkerItem`에 `sigungu` 추가
- 구 단위 묶음 기준을 API 응답 기반으로 안정화

## 5. 권장 구현 순서

1. 팝업 지도 전용 마커 크기 축소
   - 현재 커스텀 마커를 60% 정도로 줄이거나, 작은 점/캡슐형으로 변경한다.
   - 메인 지도 마커와 스타일을 분리한다.

2. 정보 카드 UX 정리
   - 마커 클릭 시 하단 카드 표시
   - 카드에는 아파트명, 최근 매매가, `이 아파트 선택` 버튼만 표시
   - 지도 빈 곳 클릭 시 카드 닫기

3. 프론트 단순 그룹핑 도입
   - API 응답에 `sigungu`가 있으면 구 단위로 묶는다.
   - `sigungu`가 없으면 우선 화면 격자 기반으로 묶는다.
   - 넓은 줌에서만 묶음 마커를 사용한다.

4. 묶음 마커 클릭 동작
   - 묶음 중심 좌표로 지도 이동
   - 줌 레벨을 1~2단계 확대
   - 바로 아파트 선택은 하지 않는다.

5. API 응답 보강 여부 결정
   - 프론트 격자 묶음으로 충분하면 백엔드 변경 없이 진행
   - 실제 구 단위 라벨이 중요하면 marker API에 `sigungu` 추가

6. QA
   - 마커가 지도 텍스트를 과도하게 가리지 않는지 확인
   - 강남/송파처럼 밀집 지역에서 겹침이 줄었는지 확인
   - 모바일/데스크톱 팝업에서 카드가 화면 밖으로 나가지 않는지 확인

## 6. 완료 기준

1. 팝업 지도에서 마커 크기가 과도하게 크지 않아야 한다.
2. 넓은 줌에서는 같은 구 또는 가까운 지역끼리 묶여 보여야 한다.
3. 가까운 줌에서는 개별 아파트를 명확히 선택할 수 있어야 한다.
4. 마커 클릭 후 카드에서 한 번 더 선택해야 부모창으로 전달된다.
5. 메인 지도 마커 디자인이 의도치 않게 깨지지 않아야 한다.
6. `npm run build`가 통과해야 한다.

## 7. 보류/검토 사항

1. 구 단위 묶음을 정확히 하려면 marker API에 `sigungu`가 필요할 수 있다.
2. 마커가 많을 때는 현재 클러스터러와 커스텀 묶음 마커 중 하나를 선택해야 한다.
3. 가격 표시가 필요한지, 아파트명 표시가 필요한지 사용자 테스트 후 결정한다.
4. 호갱노노와 유사한 집 모양 마커는 피하고, 서비스 고유의 캡슐/핀 조합으로 디자인한다.
