## 1. 배경 및 문제 정의

현재 실거래탭 사이드바는 조회 기능은 동작하지만, 아래 문제로 실사용 효율이 떨어진다.

1. 필터가 많아질수록 결과 예측이 어려워 탐색 시간이 길어진다.
2. 가격흐름이 단일 시계열 중심이라 추세/변동성 해석이 어렵다.
3. 이상치 거래가 흐름을 왜곡해 의사결정 신뢰도를 낮춘다.
4. "지금 무엇부터 구현해야 하는지" 우선순위 기준이 문서에 약하다.

## 2. 목표

### 2.1 기능 목표

1. 사용자 10초 내 핵심 필터 적용 가능
2. 기간 비교 기반 가격 해석 즉시 가능
3. 이상치 ON/OFF 비교로 흐름 신뢰도 확보
4. 구현 우선순위를 P0/P1/P2로 명확히 분리

### 2.2 지표 목표 (출시 후 2주)

1. 필터 적용 후 첫 유효 결과 도달 시간 30% 단축
2. 가격흐름 패널 상호작용률(기간/비교 토글) 25% 증가
3. 저장 필터 재사용률(7일) 20% 이상

## 3.1 필터 UI 디자인 가이드 (호갱노노 느낌)

1. 상단에 핵심 필터 칩(기간, 거래유형, 면적, 가격)을 1열로 빠르게 스캔 가능하게 배치
2. 세부 조건은 아코디언/드로어로 접어서 기본 화면 복잡도를 낮춤
3. 선택된 조건은 칩 Active 상태로 명확히 표시하고 해제(X)를 즉시 제공
4. 하단 고정 액션으로 `적용`/`초기화` 버튼을 노출해 한 손 조작 흐름을 유지
5. 결과 건수(`현재 조건 N건`)를 액션 버튼 인접 위치에 배치해 즉시 피드백 제공
6. 간격/테두리/배경 대비는 정보 밀도는 높게, 텍스트 가독성은 유지하는 톤으로 통일

## 4. 우선순위 (핵심)

## 4.1 P0 - 바로 구현 (이번 스프린트)

1. 빠른 필터 프리셋(신축/대단지/최근거래많음)
2. 기간 필터(1/3/6/12개월 + 직접 지정)
3. 직전 동기간 비교 토글 + 비교 카드(중앙값/평균/거래량/상승률)
4. 이상치 제외 토글 + 건수 즉시 반영
5. 상단 요약 카드(최근가/직전대비/거래건수)
6. "현재 조건 N건" 실시간 피드백

구현 이유: 사용자 체감 개선이 가장 크고, 백엔드 의존도가 상대적으로 낮다.

## 4.2 P1 - P0 안정화 후

1. 고급 필터(층수/연차/동 라인)
2. 저장 필터(생성/수정/삭제, 최대 10개)
3. 듀얼 시각화(실거래 점 + 평당가 선)
4. 3개월/6개월 이동평균선

구현 이유: 재방문 사용자 효율 및 가격 해석 품질 향상.

## 4.3 P2 - 데이터/정책 정렬 후

1. 가격 분포 뷰(IQR/최소/중앙/최대)
2. 이벤트 마커(정책/금리/입주)
3. 급매의심/특수거래 기준 고도화

구현 이유: 데이터 정책 합의와 운영 기준이 선행되어야 정확도 확보 가능.

## 6. 상세 요구사항

## 6.1 필터 (P0/P1)

1. 프리셋 클릭 1회 다중 조건 적용
2. 기간 필터 직접입력 시 유효성 검사(시작<=종료)
3. 이상치 제외 기준 툴팁 제공
4. 저장 필터는 이름 중복 허용, 동일 이름 구분은 id 기준

## 6.2 가격흐름 (P0/P1/P2)

1. 비교 카드 지표 단위/포맷 일관화
2. 상승률은 기준값 0 또는 null 예외 처리
3. 분포 뷰는 데이터 5건 미만일 때 "표본 부족" 노출

## 6.3 UX

1. 데스크톱 sticky, 모바일 접힘/펼침
2. 적용/초기화 버튼 모바일 하단 고정
3. 로딩 중 최근 값 유지 + 스켈레톤 노출
